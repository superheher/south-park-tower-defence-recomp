## Эта ночь (cont.151–153) — закрытие menu-render расследования

Гнал cont.150 «видимый win» = добавить реальный menu BACKGROUND под текст. **Обе тропы к фону опровергнуты измерением, placement переподтверждён заблокированным:**

- **cont.151 — нет static текстуры фона.** Menu directory-сегменты биндят ТОЛЬКО шрифтовой глиф-атлас (0xA337D000). cont.150 «texFetch=9/seg = текстуры забинжены» оказался ПЕРЕСЧЁТОМ null type-0 fetch-const'ов (гистограмма fetch[]: type-0=25 null, type-2=2 из которых декодится 1 = шрифт). Панели — плоские (cont.138). Построил pipeline декода/композита, измерил, откатил (фидил шрифт как фон = мусор), оставил measurement-only диаг.
- **cont.152 — нет декодированного видео фона.** Гипотеза «фон = attract video» (cont.139-140) не применима: VC-1 decoder ЗАСТРЯЛ — frame-pool Y-планы чёрные/uniform, U/V uniform (нет декодированного кадра).
- **cont.152 — placement тоже заблокирован.** Menu-лейблы в локальных (text-box) координатах (x≈0-72, y≈0-16); экранные позиции = per-draw World transform, который производится ТОЛЬКО на gated draw (cont.143 переподтверждён).
- **cont.153 — дочистка достижимого deliverable.** Белый-ish (low-saturation) фильтр убирает зелёные блоки (character chroma-key sprite-атлас, мис-растеренный как «лейбл») → чистый menu-текст. (Остаточный белый garble от мис-декода динамических строк — лимит cont.142, без OCR не фильтруется.)

### Стена точно: cont.21 цикл по 3 осям
Титл строит реальный per-frame рендер меню (геометрия + transform reg-0x4000 + текстуры reg-0x4800 + DRAW_INDX) в **DEFERRED device-state**, эмитит его **flush'ем** (sub_821C6D58), который **гейтится на РЕАЛЬНЫЙ GPU-completion**. У variant A нет реального рендера → нет completion → flush не срабатывает → нет реальных draws/transform'ов/фона. Цикл: титлу нужны GPU-результаты чтобы продвинуться; variant A нужно чтобы титл продвинулся чтобы их дать.
- **Фон** — нет static текстуры (cont.151) И нет video (cont.152); панели плоские.
- **Позиции** — transform только на gated draw (cont.143/152).
- **Completion** — flush gated; ФЕЙК gates (cont.149 REX_SATGATE) и фейк fence (cont.25-30) НЕ дренят (титлу нужны РЕАЛЬНЫЕ pixel-результаты: occlusion query + EDRAM resolve).

## Почему нельзя добить ПРЯМО ЗДЕСЬ
- **Прод-оракул недоступен.** A2B-план (раздел 4) опирается на сравнение с рабочим `south_park_td` под gdb (ground truth). В этом окружении прод **дедлочится на boot** (никогда не презентит) — метод prod↔variantA сравнения недоступен.
- **Read-only рычаги исчерпаны.** Нельзя ПРОЧИТАТЬ позиции/фон/completion из состояния (cont.143/151/152), нельзя ФЕЙКОМ удовлетворить completion (cont.25-30/149).

## Развилка — нужно твоё решение

1. **Deep GPU-build (multi-day; твой выбор cont.150 «grind rearchitecture»).** Реимплементировать flush: сериализовать deferred device-state → исполняемый PM4 → рендерить реальные пиксели в EDRAM (0xB0000000) → произвести viz-query (occlusion) + RB_COPY resolve результаты → удовлетворить completion → замкнуть A↔B → живое меню. Глубокая GPU-subsystem работа, многодневная, и **без прод-оракула для верификации** (поэтому я НЕ стал авто-пилотить её всю ночь — риск намолотить кучу без видимого результата).
2. **Починить прод-оракул в этом окружении** (почему south_park_td не буутит здесь) → разблокирует A2B-метод сравнения → deep build становится верифицируемым.
3. **Принять достижимый deliverable** (сплэши + menu-текст собственным движком variant A) как результат рендерер-спайка и закрыть/перенацелить.

**Моё мнение:** read-only расследование честно закрыто (cont.137-152). Дальше — только (1), и это серьёзный многодневный коммит, который стоит делать осознанно (с верификацией), а не авто-пилотом ночью. Я держу loop живым на tractable дочистке deliverable, пока ты не решишь по развилке.
